云同步（GitHub Gist）

将加密后的保险库存到你自己的 私有 GitHub Gist，实现跨设备同步。 Gist 内容始终是密文，GitHub 看不到任何明文。

在 GitHub → Settings → Developer settings → Fine-grained tokens 创建，仅需勾选 Gists: Read and write 权限。

⚠ 同步冲突

检测到远端 Gist 与本地不一致——你可能在另一台设备上修改过。请选择处理方式：

从 GitHub Gist 恢复

从另一台已启用云同步的设备接入。你需要：
1. 该设备的 Gist ID
2. 一个有 Gists 权限的 GitHub PAT
3. 创建保险库时的 主密码

📱 用配对码接入

在源设备打开「设置 → 云同步 → 📱 添加新设备」，然后：
· 扫码打开的链接会自动填入 配对码；
· 或将链接粘贴到下方，再输入源设备屏幕上显示的 4 位 PIN。

🔄 更换 Personal Access Token

粘贴新的 GitHub Fine-grained PAT（需 Gists: Read and write 权限）。 Gist ID 不会改变——新 Token 必须能访问当前 Gist，否则不予保存。

在 GitHub 设置 创建新 token。旧 token 若已泄露/过期，请到 token 列表 吊销。

⚠️ 重置保险库

此操作会清除本设备浏览器中的所有保险库数据（条目、主密码验证器、同步凭证）。 本地数据一经删除无法恢复。请选择处理方式：

本设备已启用云同步。你可以只清本地（云端保留，将来可在别处恢复）， 或同时删除云端 Gist（一同永久销毁）。

⚠️ 主密码遗忘 = 数据无法恢复，请务必牢记并定期导出备份。
